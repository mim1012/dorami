# 2026-03-07 개발 로그

## 버그 수정

### admin 유저 목록 배송지 항상 `-` 표시 문제

**파일**: `backend/src/modules/admin/admin.service.ts`

**원인**

`getUsers`에서 `$queryRaw` + `CAST("shippingAddress" AS TEXT)` 조합으로 주소를 조회했는데,
Prisma는 `Json` 컬럼에 문자열을 저장할 때 JSON 인코딩(큰따옴표 포함)해서 저장함.

`CAST AS TEXT`는 PostgreSQL의 JSON 표현 그대로 반환하므로
`"iv:authTag:ciphertext"` (바깥 따옴표 포함) 형태로 `tryDecryptAddress`에 전달됨.

`split(':')` 시 IV 값이 `"iv` (따옴표 포함) → hex 변환 실패 → 복호화 항상 `null` → 배송지 전부 `-`

**수정**

`$queryRaw` 블록 제거. Prisma가 이미 올바르게 파싱한 `user.shippingAddress`를
`formatShippingAddressSummary`에 직접 전달하는 방식으로 단순화.

```typescript
// before: $queryRaw로 별도 조회 후 tryDecryptAddress 직접 호출
// after: Prisma 조회값 그대로 formatShippingAddressSummary 사용
shippingAddressSummary: this.formatShippingAddressSummary(user.shippingAddress),
```

**확인 사항**

- 프로덕션 서버(`15.165.66.23`) `PROFILE_ENCRYPTION_KEY`는 로컬과 동일 → 키 불일치 없음
- `.env.staging`의 키는 더미값 (`0123456789abcdef...`) → staging 환경은 복호화 불가 상태 (별도 조치 필요)
