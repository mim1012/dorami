## 프로젝트 개요

### 1.1. 프로젝트 명칭
**1인 셀러 라이브 커머스 MVP 플랫폼**

### 1.2. 프로젝트 목표
1인 셀러(개인 판매자)가 라이브 방송을 통해 상품을 판매할 때 발생하는 **주문 관리의 복잡성, 재고 중복 판매, 고객 불만**을 자동화된 시스템으로 해결하는 것입니다. 사용자(시청자)는 3초 안에 가입하여 라이브 중 상품을 구매할 수 있으며, 판매자는 방송에만 집중할 수 있는 환경을 제공합니다.

### 1.3. 핵심 가치 제안 (Value Proposition)

| 대상 | 제공 가치 |
|---|---|
| **시청자/구매자** | 라이브 방송을 보며 즉시 상품을 구매할 수 있는 몰입형 쇼핑 경험 |
| **1인 셀러** | 주문 관리를 자동화하여 방송에만 집중할 수 있는 환경 |
| **플랫폼** | 높은 전환율과 사용자 만족도를 통한 지속 가능한 비즈니스 모델 |

## 문제 정의 및 솔루션

### 2.1. 현재 문제점 (Current Pain Points)

**1. 수동 주문 관리의 한계**
- 현황: 수심 명이 동시에 주문할 때 DM/댓글로 일일이 확인하고 처리해야 함.
- 결과: 주문 누락, 처리 지연, 고객 불만 발생.

**2. 재고 중복 판매 (Overselling)**
- 현황: 실시간 재고 차감이 불가능하여 같은 상품을 여러 고객에게 판매하는 상황 발생.
- 결과: 환불 처리, 고객 신뢰도 하락.

**3. 고객 불만 증가**
- 현황: 선착순 기준이 모호하여 공정성 시비 발생.
- 결과: 고객 이탈, 부정적 리뷰.

### 2.2. 제안된 솔루션 (Project Goals)

**1. 주문 처리 100% 자동화**
- 라이브 중 발생하는 모든 주문을 시스템이 자동으로 처리합니다.
- 판매자는 주문 관리에 신경 쓰지 않고 방송에만 집중할 수 있습니다.

**2. 한정 수량/예비번호 자동 로직**
- 관리자 개입 없이 시스템이 자동으로 재고를 관리하고 예비번호를 부여합니다.
- 공정하고 투명한 주문 처리로 고객 신뢰도 향상.

**3. 1인 운영 최적화**
- 방송 진행과 동시에 판매 관리가 가능한 환경을 제공합니다.
- 별도의 관리자 없이 1인이 모든 것을 처리할 수 있습니다.

## 디자인 철학

### 3.1. 핵심 원칙

**몰입 우선 (Immersive-First)**
- 사용자의 시선이 라이브 영상에 집중되도록, UI는 필요할 때만 나타나고 영상 콘텐츠를 방해하지 않아야 합니다.
- 라이브 화면에서는 영상이 화면의 90% 이상을 차지하며, 상품 정보와 채팅은 반투명 배경 위에 배치됩니다.

**명확성과 단순성 (Clarity & Simplicity)**
- 모든 정보와 기능은 명확하고 이해하기 쉬워야 합니다.
- 복잡한 과정은 과감히 제거하고, 사용자가 고민 없이 행동할 수 있도록 유도합니다.
- 로그인은 3초 안에, 구매는 최소 3번의 터치로 완료할 수 있어야 합니다.

**강렬한 시각적 단서 (Strong Visual Cues)**
- "Hot Pink" 액센트 컬러를 전략적으로 사용하여 구매, 알림 등 핵심적인 행동을 유도합니다.
- 긴급한 정보(재고 임박, 타이머 만료 임박 등)를 효과적으로 전달합니다.

### 3.2. 참고 모델
- **Reference**: Spring Day Live (springday-live.com)
- **적용 요소**: 모바일 최적화, 라이브 중심 인터페이스, 실시간 상품 정보, 간편한 구매 프로세스

## 화면별 상세 설계

### 5.1. 로그인 화면 (AUTH-01)

**목표**: 사용자가 3초 안에 최소한의 정보 입력으로 회원가입 및 로그인을 완료.

**핵심 요소**:
- 로고 (상단)
- "카카오로 계속하기" 버튼 (Primary CTA, 화면 중앙)
- "인스타그램으로 계속하기" 버튼 (Secondary)
- 이메일/비밀번호 입력 필드 (선택적)

**UX 플로우**:
1. 사용자가 "카카오로 계속하기" 터치
2. 카카오 SDK를 통해 인증 및 프로필 정보 요청
3. 성공 시 홈 화면으로 이동
4. 실패 시 에러 메시지 토스트로 표시

### 5.2. 메인 홈 화면 (HOME-01)

**목표**: 예정된 라이브 방송에 대한 기대감을 높이고, 방송이 없는 동안에도 상품 탐색 유도.

**핵심 요소**:
- 라이브 카운트다운 배너 (상단, Display 타이포그래피)
- "알림받기" 버튼 (Hot Pink)
- 매거진 스타일 상품 큐레이션 카드
- 하단 탭 바

**디자인 특징**:
- 비대칭 그리드 레이아웃으로 시각적 즐거움 제공
- 카드 터치 시 미세한 줌인 효과
- 라이브 진행 중이면 카운트다운 배너가 "LIVE NOW"로 변경되고, 터치 시 라이브 화면으로 즉시 이동

### 5.3. 메인 라이브 화면 (LIVE-01)

**목표**: 영상 시청을 방해받지 않으면서, 실시간으로 소통하고 즉시 상품을 구매할 수 있는 몰입형 환경.

**핵심 요소**:
- 풀스크린 라이브 비디오 플레이어 (화면의 90% 이상)
- 실시간 채팅창 (하단, 반투명 배경)
- 상품 정보 캐러셀 (하단, 좌우 스와이프)
- 플로팅 액션 버튼 (공유, 좋아요)

**기술 요구사항**:
- WebSocket을 통한 실시간 채팅 및 재고 업데이트
- HLS/DASH 스트림 재생
- 상품 캐러셀에서 상품 터치 → 옵션 선택 모달 표시

### 5.4. 상품 옵션 선택 모달 (PROD-01)

**목표**: 사용자가 복잡한 과정 없이 옵션을 빠르게 선택하고 장바구니에 담기.

**핵심 요소**:
- 상품명 및 가격
- 옵션 선택 그룹 (색상, 사이즈 등)
- 수량 조절 버튼
- "장바구니 담기" CTA 버튼

**UX 특징**:
- Bottom Sheet 형태의 모달로 라이브 화면과의 연속성 유지
- 품절 옵션은 Secondary Text 색상과 취소선으로 구분
- 모든 필수 옵션 선택 전까지 CTA 버튼 비활성화

### 5.5. 장바구니 및 타이머 화면 (CART-01)

**목표**: 10분이라는 시간제한을 통해 빠른 구매 결정 유도 및 FOMO 심리 자극.

**핵심 요소**:
- 10분 카운트다운 타이머 바 (상단, Hot Pink 프로그레스 바)
- 장바구니 상품 목록
- 총 결제 금액
- "결제하기" CTA 버튼

**디자인 특징**:
- 시간이 줄어들수록 타이머 바가 붉은색으로 변하며 깜빡이는 효과
- 시간 초과 시 자동으로 장바구니 비우기 및 알림 표시

### 5.6. 주문 완료 화면 (ORDER-01)

**목표**: 주문 성공을 명확히 알리고, 무통장 입금에 필요한 정보를 정확하게 전달하여 오입금 방지.

**핵심 요소**:
- 주문 완료 아이콘 및 메시지 (중앙, 큰 체크 아이콘)
- 주문 번호
- 입금 정보 (은행, 계좌번호, 예금주)
- 최종 결제 금액 및 입금 기한
- "주문내역 보기" 버튼
- "카카오톡으로 받기" 버튼

**UX 특징**:
- 입금 정보는 별도 카드 컴포넌트로 그룹화
- 계좌번호와 같은 중요 정보는 Heading 2 스타일로 강조
- 입금 기한은 명확한 포맷으로 표시 (예: "2026년 1월 24일 23시 59분까지")

## 프론트엔드 기술 스택

### 6.1. 핵심 기술

| 구분 | 기술 | 선정 사유 |
|---|---|---|
| Core Framework | React Native (with Expo) | 단일 코드베이스로 iOS/Android 동시 개발, 빠른 MVP 출시 |
| Language | TypeScript | 타입 안정성, 코드 품질 향상 |
| State Management | Zustand | 가볍고 직관적, 보일러플레이트 최소화 |
| Styling | Styled Components | 컴포넌트 기반 스타일링, 다크 모드 구현 용이 |
| Navigation | React Navigation | 복잡한 사용자 플로우 안정적 구현 |
| Data Fetching | React Query | 서버 데이터 관리 자동화 |
| Real-time Comms | Socket.IO Client | WebSocket 기반 실시간 기능 |

### 6.2. 프로젝트 구조

```
/src
├── /api                    # API 호출 함수
├── /assets                 # 폰트, 이미지 등
├── /components             # UI 컴포넌트
│   ├── /common            # 범용 컴포넌트
│   └── /domain            # 도메인 특화 컴포넌트
├── /constants             # 상수 값
├── /hooks                 # 커스텀 훅
├── /navigation            # 내비게이션 설정
├── /screens               # 각 화면 컴포넌트
├── /store                 # Zustand 스토어
├── /styles                # 전역 스타일, 테마
├── /types                 # TypeScript 타입
└── /utils                 # 유틸리티 함수
```

### 6.3. 핵심 기능 구현 가이드

**디자인 시스템 구현**
- `/src/styles/theme.ts`에 색상 팔레트, 타이포그래피 스케일을 객체로 정의
- Styled Components의 `ThemeProvider`로 앱 전체에 테마 주입
- 모든 컴포넌트에서 `theme.colors.hotPink` 형태로 디자인 토큰 접근

**실시간 라이브 방송 화면**
- `react-native-video` 또는 Expo의 `AV` 모듈로 HLS/DASH 스트림 재생
- Socket.IO 클라이언트로 채팅, 재고 업데이트 실시간 처리
- 화면 진입 시 소켓 연결, 진출 시 정리(cleanup)

**장바구니 타이머 로직**
- 타이머 기준: 반드시 서버 시간 사용 (클라이언트 시간 신뢰 불가)
- 백엔드에서 `expiresAt` 전달, 클라이언트에서 차이 계산
- 앱 백그라운드 전환 후 재활성화 시에도 정확한 시간 계산
