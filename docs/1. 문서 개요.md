_# 1인 셀러 라이브 커머스 MVP: 화면별 상세 UI/UX 설계서

## 1. 문서 개요

본 문서는 "1인 셀러 라이브 커머스 MVP 플랫폼"의 모바일 애플리케이션 각 화면에 대한 상세한 UI(사용자 인터페이스) 및 UX(사용자 경험) 설계 명세를 정의합니다. 디자이너와 프론트엔드 개발자는 이 문서를 기반으로 일관되고 높은 품질의 사용자 경험을 구현해야 합니다.

- **참고 문서**: `ppt_analysis.md`, `design_system.md`

## 2. 전역(Global) UI/UX 요소

### 2.1. 상태 표시줄 (Status Bar)
- **스타일**: 다크 모드 콘텐츠에 맞춰 항상 라이트 스타일(흰색 아이콘 및 텍스트)을 유지합니다.
- **배경**: 앱의 배경색(`Primary Black`)과 자연스럽게 어우러지도록 투명하게 처리합니다.

### 2.2. 하단 탭 바 (Bottom Navigation)
- **역할**: 앱의 핵심 기능(Home, Shop, Live, Messages, My Page) 간의 빠른 전환을 제공하는 주 내비게이션입니다.
- **디자인 관점**:
    - `design_system.md`에 정의된 탭 바 컴포넌트 명세를 따릅니다.
    - 선택된 탭의 아이콘과 레이블은 `Hot Pink` 색상으로, 비활성 탭은 `Secondary Text` 색상으로 표시하여 현재 위치를 명확히 인지시킵니다.
- **프론트엔드 관점**:
    - 각 탭은 독립적인 내비게이션 스택을 가질 수 있어야 합니다. (예: Shop 탭 내에서 상품 상세 페이지로 이동해도 탭 바는 유지)
    - 라이브 방송 중에는 탭 바를 숨겨 영상에 대한 몰입감을 극대화하는 것을 고려할 수 있습니다.

## 3. 화면별 상세 설계

### 3.1. 로그인 화면 (ID: AUTH-01)

- **화면 목표**: 사용자가 3초 안에 최소한의 정보 입력으로 회원가입 및 로그인을 완료하게 한다.

| 구분 | 설계 명세 |
|---|---|
| **핵심 컴포넌트** | - 로고<br>- "카카오로 계속하기" 버튼<br>- "인스타그램으로 계속하기" 버튼<br>- 이메일/비밀번호 입력 필드 (선택적) |
| **디자이너 관점** | - 제안서의 "EASY ACCESS" 디자인을 기반으로, 앱의 로고를 상단에 배치하여 브랜드 아이덴티티를 강조합니다.<br>- 소셜 로그인 버튼을 화면 중앙에 크게 배치하여 사용자의 첫 행동을 명확하게 유도합니다.<br>- 전체적으로 `Dark Mode & Hot Pink` 테마를 적용하여 강렬한 첫인상을 제공합니다. |
| **프론트엔드 관점** | - **User Flow**: "카카오로 계속하기" 터치 → 카카오 SDK를 통해 인증 및 프로필 정보(이름, 이메일) 요청 → 성공 시 홈 화면으로 이동, 실패 시 에러 메시지 토스트로 표시.<br>- **상태 관리**: API 요청 중에는 버튼을 비활성화하고 로딩 스피너를 표시합니다.<br>- **데이터**: 카카오로부터 받은 사용자 정보를 서버로 전송하여 회원가입/로그인 처리를 완료합니다. |

### 3.2. 메인 홈 화면 (ID: HOME-01)

- **화면 목표**: 예정된 라이브 방송에 대한 기대감을 높이고, 방송이 없는 동안에도 상품을 탐색하며 쇼핑 경험을 이어가게 한다.

| 구분 | 설계 명세 |
|---|---|
| **핵심 컴포넌트** | - 라이브 카운트다운 배너<br>- "알림받기" 버튼<br>- 매거진 스타일 상품 큐레이션 카드<br>- 하단 탭 바 |
| **디자이너 관점** | - 화면 최상단에 다음 라이브까지 남은 시간을 보여주는 카운트다운 타이머(`Display` 타이포그래피 적용)를 배치하여 시각적 긴장감을 조성합니다.<br>- 상품 목록은 단순 나열이 아닌, 이미지 중심의 비대칭 그리드 레이아웃(매거진 스타일)으로 구성하여 시각적 즐거움을 더합니다.<br>- 각 상품 카드는 마우스를 올렸을 때(PC 웹의 경우) 또는 터치했을 때 미세한 줌인 효과를 주어 인터랙티브한 느낌을 강화합니다. |
| **프론트엔드 관점** | - **데이터**: 서버로부터 다음 라이브 방송 정보(시간, 상품)와 큐레이션 상품 목록 데이터를 받아와 렌더링합니다.<br>- **상태 관리**: 라이브 방송이 진행 중일 경우, 카운트다운 배너는 "LIVE NOW" 상태로 변경되며, 터치 시 라이브 화면(LIVE-01)으로 즉시 이동합니다.<br>- **인터랙션**: "알림받기" 버튼 클릭 시 푸시 알림 권한을 요청하고, 서버에 알림 등록 API를 호출합니다. |

### 3.3. 메인 라이브 화면 (ID: LIVE-01)

- **화면 목표**: 사용자가 영상 시청을 방해받지 않으면서, 실시간으로 소통하고 즉시 상품을 구매할 수 있는 몰입형 쇼핑 환경을 제공한다.

| 구분 | 설계 명세 |
|---|---|
| **핵심 컴포넌트** | - 풀스크린 라이브 비디오 플레이어<br>- 실시간 채팅창<br>- 상품 정보 캐러셀<br>- 플로팅 액션 버튼 (공유, 좋아요) |
| **디자이너 관점** | - 화면의 90% 이상을 비디오 플레이어에 할당하고, UI 요소들은 반투명 배경 위에 배치하여 영상 콘텐츠를 최대한 가리지 않도록 합니다.<br>- 하단 상품 캐러셀은 좌우 스와이프로 탐색 가능하며, 현재 방송에서 소개되는 상품을 시각적으로 강조(예: `Hot Pink` 테두리)합니다.<br>- 사용자가 보내는 채팅 메시지와 시스템 메시지(예: "OOO님이 입장했습니다")는 시각적으로 구분하여 가독성을 높입니다. |
| **프론트엔드 관점** | - **기술 스택**: WebSocket을 사용하여 채팅 메시지와 실시간 시청자 수, 재고 변동 등을 지연 없이 업데이트합니다. 비디오 스트리밍은 HLS 또는 WebRTC 프로토콜을 사용합니다.<br>- **인터랙션**: 상품 캐러셀에서 상품 터치 → 상품 옵션 선택 모달(PROD-01) 표시.<br>- **애니메이션**: 새로운 채팅이 들어올 때 아래에서 위로 부드럽게 스크롤되는 애니메이션을 적용합니다. |

### 3.4. 상품 옵션 선택 모달 (ID: PROD-01)

- **화면 목표**: 사용자가 복잡한 과정 없이 색상, 사이즈 등 원하는 옵션을 빠르게 선택하고 장바구니에 담게 한다.

| 구분 | 설계 명세 |
|---|---|
| **핵심 컴포넌트** | - 상품명 및 가격<br>- 옵션 선택 그룹 (예: 색상, 사이즈)<br>- 수량 조절 버튼<br>- "장바구니 담기" CTA 버튼 |
| **디자이너 관점** | - 화면 하단에서 부드럽게 올라오는 Bottom Sheet 형태의 모달로 제공하여, 라이브 화면과의 연속성을 유지합니다.<br>- 선택 가능한 옵션은 `Content Background` 위에 `Primary Text`로, 품절된 옵션은 `Secondary Text` 색상과 취소선을 사용하여 명확히 구분합니다.<br>- "장바구니 담기" 버튼은 `Hot Pink` 색상으로 강력하게 행동을 유도하며, 모든 옵션이 선택되기 전까지는 비활성화 상태로 둡니다. |
| **프론트엔드 관점** | - **상태 관리**: 각 옵션 그룹(색상, 사이즈)의 선택 상태를 관리합니다. 모든 필수 옵션이 선택되면 CTA 버튼을 활성화합니다.<br>- **데이터**: 실시간 재고 데이터를 기반으로 각 옵션의 품절 여부를 표시합니다. 사용자가 옵션을 선택할 때마다 해당 조합의 재고를 다시 확인하여 "품절 임박" 등의 메시지를 표시할 수 있습니다.<br>- **인터랙션**: "장바구니 담기" 버튼 터치 → 장바구니에 상품 추가 API 호출 → 성공 시 모달을 닫고 "장바구니에 상품을 담았습니다" 토스트 메시지 표시. |

### 3.5. 장바구니 및 타이머 화면 (ID: CART-01)

- **화면 목표**: 10분이라는 시간제한을 통해 사용자의 빠른 구매 결정을 유도하고, FOMO(Fear of Missing Out) 심리를 자극한다.

| 구분 | 설계 명세 |
|---|---|
| **핵심 컴포넌트** | - 10분 카운트다운 타이머 바<br>- 장바구니 상품 목록<br>- 총 결제 금액<br>- "결제하기" CTA 버튼 |
| **디자이너 관점** | - 화면 최상단에 `Hot Pink` 색상의 프로그레스 바 형태 타이머를 배치하여 시간의 흐름을 시각적으로 명확하게 전달합니다.<br>- 시간이 줄어들수록(예: 1분 미만) 타이머 바가 붉은색 계열로 변하며 깜빡이는 효과를 주어 긴박감을 고조시킵니다.<br>- 상품 목록은 간결하게 상품 이미지, 이름, 가격, 수량 정보만 표시하여 사용자가 결제에 집중할 수 있도록 합니다. |
| **프론트엔드 관점** | - **로직**: 장바구니에 상품이 담긴 순간부터 10분의 타이머가 시작됩니다. 타이머는 브라우저/앱을 벗어나도 서버 시간을 기준으로 유지되어야 합니다.<br>- **인터랙션**: "결제하기" 버튼 터치 → 주문/결제 화면으로 이동. 시간이 만료되면 "시간이 초과되어 상품이 장바구니에서 삭제되었습니다" 알림을 표시하고, 장바구니를 비우는 API를 호출합니다.<br>- **상태 관리**: 여러 상품이 담겨 있을 경우, 각 상품별 타이머가 아닌 장바구니 전체에 대한 단일 타이머를 적용하여 UX를 단순화합니다. |

### 3.6. 주문 완료 화면 (ID: ORDER-01)

- **화면 목표**: 사용자에게 주문이 성공적으로 완료되었음을 명확히 알리고, 무통장 입금에 필요한 정보를 정확하게 전달하여 오입금을 방지한다.

| 구분 | 설계 명세 |
|---|---|
| **핵심 컴포넌트** | - 주문 완료 아이콘 및 메시지<br>- 주문 번호<br>- 입금 정보 (은행, 계좌번호, 예금주)<br>- 최종 결제 금액 및 입금 기한<br>- "주문내역 보기" 버튼 |
| **디자이너 관점** | - 화면 중앙에 큰 체크 아이콘과 "주문이 완료되었습니다" 메시지를 배치하여 긍정적인 피드백을 즉각적으로 제공합니다.<br>- 입금 정보는 별도의 카드 컴포넌트 안에 그룹화하고, 계좌번호와 같이 중요한 정보는 `Heading 2` 스타일로 강조하여 가독성을 높입니다.<br>- 입금 기한은 "OO년 OO월 OO일 OO시까지"와 같이 명확한 포맷으로 표시하고, 남은 시간을 함께 보여주어 입금을 유도합니다. |
| **프론트엔드 관점** | - **데이터**: 결제 완료 후 서버로부터 받은 주문 상세 정보(주문번호, 결제금액, 가상계좌 정보 등)를 표시합니다.<br>- **인터랙션**: "카카오톡으로 받기" 버튼(제안서 내용)을 구현하여, 터치 시 서버에 카카오 알림톡 발송을 요청하는 API를 호출합니다.<br>- **후속 조치**: 사용자가 이 화면을 벗어나도, 마이페이지의
